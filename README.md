# Eye Tracking Studio

Учебная демонстрационная система слежения за взглядом на Python. Программа использует обычную веб-камеру, находит лицо, глаза и радужку через MediaPipe Face Landmarker, оценивает направление взгляда и показывает результат в desktop-окне `Eye Tracking Studio` с обычными элементами интерфейса Windows.

Проект можно использовать как основу для курсовой или практической работы по теме **"Реализация системы слежения за взглядом (eye tracking)"**.

## Цель проекта

Показать, как с помощью компьютерного зрения можно приблизительно определить направление взгляда пользователя без специального оборудования. Система не претендует на точность медицинского или промышленного eye tracker, но демонстрирует основные этапы такой технологии:

- захват видеопотока с веб-камеры;
- поиск лица, глаз и радужки;
- вычисление относительного положения радужки внутри глаза;
- сглаживание результата по нескольким кадрам;
- калибровка под пользователя и экран;
- визуализация направления взгляда;
- запись измерений для дальнейшего анализа.

## Анализ изображений

Помимо live-режима программа умеет искать взгляд на статичных изображениях. Для этого можно передать один файл или папку с картинками:

```bash
python eye_tracking.py --image photo.jpg --preview
python eye_tracking.py --image-dir samples --output-dir output/gaze
```

В этом режиме приложение:

- находит лицо, глаз и радужку на каждой картинке;
- оценивает направление взгляда;
- сохраняет размеченную копию изображения с подписью и маркером взгляда;
- для папки обрабатывает все подходящие изображения по очереди.

Если `--output-dir` не указан, результат сохраняется рядом с исходным файлом с суффиксом `_gaze.png`.

## Возможности

- Отображение видеопотока в desktop-интерфейсе с меню, кнопками, чекбоксами, ползунками и статусной строкой.
- Определение направления взгляда: центр, вверх, вниз, влево, вправо и диагональные направления.
- Отображение confidence, количества найденных глаз и мини-карты взгляда.
- Сглаживание направления взгляда по истории последних кадров.
- Калибровка по точкам экрана.
- Управление курсором сразу после запуска, без калибровки.
- Выбор алгоритма слежения за взглядом: `threshold`, `majority`, `adaptive`, `ema` и `hysteresis`.
- Выбор алгоритма слежения за взглядом прямо в графическом интерфейсе.
- Анализ направления взгляда на статичных изображениях.
- Запись измерений в CSV-файл.
- Автоматическая загрузка модели `face_landmarker.task` при первом запуске.

## Другие алгоритмы слежения за взглядом

В проекте можно использовать несколько разных подходов к распознаванию взгляда:

- `threshold` - самый быстрый и прямой режим: сразу превращает текущие координаты взгляда в направление без истории.
- `majority` - режим голосования: смотрит на несколько последних кадров и выбирает то направление, которое встречалось чаще всего.
- `adaptive` - режим подстройки: работает как `majority`, но еще медленно смещает центр калибровки, если пользователь долго смотрит в центр.
- `ema` - режим экспоненциального сглаживания: сначала сглаживает сами координаты взгляда, а потом классифицирует их. Полезен, когда направление прыгает из-за шума.
- `hysteresis` - режим гистерезиса: не переключает направление на новое мгновенно, а ждет устойчивого подтверждения. Хорош для борьбы с дрожанием между соседними зонами.

### Как это работает в коде

Сначала программа получает координаты `ratio_x` и `ratio_y` для каждого глаза, а потом уже переводит их в направление:

1. `estimate_eye_from_landmarks()` находит радужку внутри области глаза и считает относительные координаты.
2. `average_gaze()` объединяет оба глаза и отбрасывает совсем слабые сигналы.
3. `GazeEstimator.classify()` превращает координаты в базовое направление `центр / вверх / вниз / влево / вправо`.
4. `GazeEstimator.analyze()` применяет выбранный режим стабилизации.
5. Результат попадает в интерфейс как `stable_direction`.

Разница между режимами именно в шаге `analyze()`:

- `threshold` берет результат `classify()` без дополнительных преобразований.
- `majority` выбирает самое частое направление из последних кадров.
- `adaptive` делает то же самое, но еще подстраивает центр калибровки.
- `ema` сначала сглаживает сами координаты, а потом снова классифицирует их.
- `hysteresis` требует устойчивого подтверждения, прежде чем переключить направление.

Отдельно `CursorController` уже использует эти координаты для движения мыши, но это другой слой логики, не алгоритм распознавания взгляда.
- десктопный интерфейс на `tkinter` для живой работы с камерой;
- старый интерфейс поверх OpenCV с HUD и горячими клавишами;
- режим анализа отдельных изображений и папок без непрерывного видеопотока;
- реализация на основе MediaPipe и обычной веб-камеры без специального оборудования;
- более точные решения с инфракрасной камерой или специализированным eye tracker-устройством.

## Установка

Рекомендуется Python 3.10-3.12. На Python 3.13 пакет `mediapipe` может быть недоступен через pip.

Создание виртуального окружения:

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

Если виртуальное окружение не нужно:

```bash
pip install -r requirements.txt
```

## Запуск

Обычный запуск:

```bash
python eye_tracking.py
```

По умолчанию открывается desktop-интерфейс на `tkinter`: окно камеры, меню `File / Tracking / Help`, панель управления, настройки сглаживания и статусы трекинга.

Если основная камера не открывается:

```bash
python eye_tracking.py --camera 1
```

Запуск с записью результатов:

```bash
python eye_tracking.py --log gaze_log.csv
```

Запуск с включенным режимом управления курсором:

```bash
python eye_tracking.py --control-cursor
```

Если нужен другой алгоритм слежения за взглядом, можно задать его как стартовый вариант:

```bash
python eye_tracking.py --gaze-algorithm adaptive
python eye_tracking.py --gaze-algorithm ema
python eye_tracking.py --gaze-algorithm hysteresis
```

После запуска алгоритм можно менять в самом окне программы в панели `Управление`.

Запуск старого OpenCV-интерфейса с HUD поверх видео:

```bash
python eye_tracking.py --opencv-ui
```

## Управление

- `C` - запустить калибровку по точкам экрана.
- `M` - включить или выключить управление курсором.
- `Ctrl+Shift+M` - выключить управление курсором.
- `R` - сбросить калибровку и модель курсора.
- `Q` - выйти из программы.

Во время калибровки нужно смотреть точно в центр целевой точки и держать голову как можно ровнее. Программа начинает брать образцы только когда последние измерения взгляда стали стабильными, поэтому на каждой точке лучше не переводить взгляд до перехода к следующему маркеру.

После изменений настроек курсора или алгоритма калибровку нужно пройти заново: нажми `R`, затем `C`.

По умолчанию калибровка стала короче: `--calibration-samples` теперь равен `6`, а не `10`.

Если курсор дрожит, можно увеличить параметр `--cursor-smoothing` или пройти калибровку заново. Если курсор слишком медленно догоняет взгляд, наоборот уменьши сглаживание:

```bash
python eye_tracking.py --cursor-smoothing 0.90
python eye_tracking.py --cursor-smoothing 0.70
```

## Параметры командной строки

| Параметр | Назначение |
| --- | --- |
| `--camera` | индекс камеры, обычно `0` или `1` |
| `--log` | путь к CSV-файлу для записи измерений |
| `--gaze-algorithm` | выбрать стартовый алгоритм слежения за взглядом: `threshold`, `majority`, `adaptive`, `ema` или `hysteresis` |
| `--control-cursor` | включить управление курсором сразу, без калибровки |
| `--cursor-mode` | выбрать стартовый алгоритм курсора: `direct`, `relative` или `calibrated` |
| `--cursor-smoothing` | сглаживание движения курсора от `0.0` до `0.95` |
| `--cursor-min-confidence` | минимальная уверенность для движения курсора |
| `--calibration-samples` | количество стабильных измерений на одну точку калибровки |
| `--opencv-ui` | запустить старый интерфейс OpenCV с наложением HUD |

## Как работает алгоритм

1. Кадр с камеры зеркально отражается, чтобы движение выглядело естественно для пользователя.
2. Кадр передается в MediaPipe Face Landmarker.
3. MediaPipe находит landmarks лица, контуры глаз и landmarks радужки.
4. Для каждого глаза вычисляется центр радужки.
5. Центр радужки переводится в относительные координаты внутри области глаза.
6. Показания двух глаз усредняются с учетом confidence.
7. Полученные координаты сравниваются с калиброванным центральным положением.
8. Направление взгляда сглаживается по нескольким последним кадрам.
9. После экранной калибровки относительные координаты взгляда преобразуются в координаты экрана.

## Сценарий демонстрации

Для защиты или показа проекта удобно использовать такой порядок:

1. Запустить программу командой `python eye_tracking.py`.
2. Показать, что система находит глаза и отображает направление взгляда.
3. Посмотреть в разные стороны и показать изменение `CURRENT GAZE`.
4. Нажать `C` и пройти калибровку по точкам.
5. Включить управление курсором клавишей `M`.
6. Запустить режим логирования и показать CSV-файл с результатами.

## Ограничения

Это демонстрационная система. На точность влияют:

- освещение;
- качество веб-камеры;
- очки и блики;
- расстояние до экрана;
- поворот головы;
- разрешение камеры;
- стабильность положения пользователя.

Для более точной версии нужны инфракрасная подсветка, специализированная камера или обученная модель, рассчитанная именно на оценку точки взгляда.

## Зависимости

Основные библиотеки:

- `opencv-python` - работа с камерой и отображение интерфейса;
- `numpy` - вычисления и аппроксимация экранной калибровки;
- `mediapipe` - поиск лица, глаз и радужки.

При первом запуске приложение использует модель `face_landmarker.task` из `.cache/mediapipe`. Если файла нет, программа попробует скачать его автоматически.
